# 이정표농업 에이전트 워크플로우

## 사용자 역할

- 사용자는 총괄 디렉터 역할을 맡는다.
- 목표, 우선순위, 수정 방향을 지시한다.
- 세부 구현은 각 에이전트가 분담 수행한다.

## 기본 에이전트

### 1. 기획 총괄 에이전트

- 무엇을 먼저 만들지 결정
- 주요 고객군 정의
- 수익 모델과 광고 정책 정리
- MVP 범위 통제

### 2. 정보 수집 에이전트

- 정부, 도, 시, 군, 농업기술센터 출처 목록 확보
- 수집 주기와 원문 링크 관리
- 신규 공고 여부 탐지

### 3. 정보 정리 에이전트

- 지원 대상, 지역, 분야, 신청 기간, 지원 금액 구조화
- 중복 공고 정리
- 사람이 읽기 쉬운 요약 작성

### 4. 추천 로직 에이전트

- 고객 프로필 기반 적합도 산정
- 왜 추천되었는지 설명 문구 생성
- 광고 기업과의 연관도 계산

### 5. 풀스택 구현 에이전트

- 프론트엔드 화면 구현
- API 및 DB 설계
- 관리자 기능과 광고 신청 기능 구현

### 6. 보안 운영 에이전트

- 관리자 로그인 보안
- 입력값 검증
- 개인정보 최소수집 정책 적용
- 로그와 권한 통제

## 추천 작업 순서

1. 기획 총괄 에이전트가 MVP 화면과 데이터 항목을 확정한다.
2. 정보 수집 에이전트가 출처와 원문 데이터를 확보한다.
3. 정보 정리 에이전트가 표준 데이터 구조로 정리한다.
4. 추천 로직 에이전트가 사용자 조건별 추천 규칙을 만든다.
5. 풀스택 구현 에이전트가 사용자 화면과 관리자 기능을 구현한다.
6. 보안 운영 에이전트가 배포 전 점검을 수행한다.
